## Summary

### CRITICAL ISSUES (3)
1. **`CK_CorrectiveAction_Status` check constraint never migrated** — live DB has `('Open', 'InProgress', 'Closed')`, model has `('Open', 'InProgress', 'Closed', 'Voided', 'Overdue')`. A migration must `DropCheckConstraint` / `AddCheckConstraint` to apply the full allowed set. Any attempt to set `Status = 'Voided'` or `'Overdue'` will throw a DB constraint violation at runtime.
2. **`Phase1C_DivisionScoreTarget` is an empty migration shell** — `Division.ScoreTarget` column has no DDL in Up(). Fresh database builds will be missing this column entirely, causing runtime failures on any query that touches `ScoreTarget`.
3. **`FindingPhoto` has `IsDeleted` (extends AuditableEntity) but no `HasQueryFilter`** — soft-deleted photos will be returned by all queries, violating the soft-delete pattern established across all other audit entities.

### ADVISORY ISSUES (8)
1. **`TemplateChangeLog` is never written to** — template change audit trail is silent; no handler populates it.
2. **`CaNotificationLog` is never written to** — CA notification deduplication table is empty; reminder emails may send multiple times per CA per day.
3. **`CorrectiveAction.AuditResponseId`** and **`AuditResponse.SortOrderSnapshot`** are declared in models but never assigned by any handler.
4. **11+ missing FK indexes** on `AuditFinding.AuditId`, `AuditFinding.QuestionId`, `AuditAttachment.AuditId`, `CorrectiveAction.AuditId`, `CorrectiveAction.QuestionId`, `AuditSection.TemplateVersionId`, `AuditTemplate.DivisionId`, `TemplateChangeLog.TemplateVersionId`, and others — will cause table scans on these FK lookups.
5. **`CorrectiveAction.Source` has no check constraint** — `"Manual"` / `"AutoGenerated"` enforced only at application layer.
6. **`Phase1B_WeightBackfill` Down() is empty** — rollback will not revert weight values; acceptable for a backfill but should be documented.
7. **`GetAllUsers` and `GetUserList` are duplicate handlers** — same authorization, same logic, both unbounded; consolidate or add paging.
8. **Unbounded `ToListAsync()` calls** in `GetSectionTrends`, export handlers, and user list queries — no hard row caps; risk at scale.
