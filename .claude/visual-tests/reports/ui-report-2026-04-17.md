## 3. Dashboard Deep-Dive (Primary Complaint: Overwhelming / Too Much Scrolling)

### 3b. Root Causes of Overwhelm

1. **Division Health cards expand vertically with division count.** With 9 divisions at 3 cards per row, the grid alone is ~350–420px and is the very first thing shown. On mobile (375px) the 2-column layout stretches it to ~900px — effectively the entire first screen is just this one section.

2. **Two separate "per-division" sections back-to-back.** Division Health (item 5) and Conformance by Division chart (item 11) both display per-division data. They are separated by 5 other sections, making the page feel repetitive and disjointed.

3. **Six header buttons create a cramped, scan-heavy toolbar.** "Report Composer", "Generate PDF", "Export" (split button), "By Employee", "Quarterly", and "Refresh" all render in a single row. At 1440px they barely fit. At 768px the "By Employee" and "Quarterly" buttons hide (correct), but the remaining 4 still feel heavy. A user visiting the dashboard to check status doesn't need report-generation actions as the primary affordance.

4. **KPI cards are buried below Division Health + Recent Audits.** The 4 headline KPIs (Total Audits, Avg Conformance, Non-Conformances, Warnings) — the most important at-a-glance numbers — are positioned at item 8, after ~700–900px of other content. SafetyCulture and Cority both lead with 3–5 large KPI tiles as the very first element, with charts below them.

5. **Section NC Breakdown cards (item 10) duplicate information already in the "Conformance by Division" chart (item 11) and the "NC by Section" chart inside the Analysis tab (item 14).** Three representations of section-level NC data on one page is the single biggest source of redundancy.

6. **The tab strip appears after the user has already scrolled past most of the content.** This defeats the purpose of tabbed navigation — tabs should reduce scroll, not be placed at the bottom of a long pre-tab scroll.

7. **Recent Audits feed (item 6) belongs in a tab, not as a permanent above-the-fold section.** It is a list view, not a summary, and competes visually with the Division Health cards directly above it.

8. **No collapsed default state.** Every section renders fully expanded on load. Sections with collapse/expand controls (the eye-slash icon) are opt-in hidden per user, but state is not persisted — they re-expand on every page load.

### 3c. Competitor Benchmark

**SafetyCulture (iAuditor):** Dashboard leads with 4 hero KPI tiles (Total Inspections, Avg Score, Incomplete, Flagged Items). Division/site breakdown is a secondary section below. Charts are behind a "Reports" navigation item, not on the main dashboard. No corrective action tables on the summary view.

**Cority:** Uses a "command center" model — configurable widget cards, each collapsible, with a default layout of KPI row → single key chart → action list. Detailed data tables are never shown on the dashboard; they link out to dedicated list views.

**Intelex:** Dashboard is KPIs + one trend sparkline + a notifications panel. Everything above the fold. No charts on the dashboard itself.

**Gap summary:** The Stronghold dashboard tries to combine a summary dashboard, a charts/reporting page, AND a data table all in one scroll — a common anti-pattern for enterprise safety software. Competitors separate these concerns into distinct navigation items.

## 4. Page-by-Page Issues

### Dashboard (`/audit-management/reports`)
**Measured: 3,614px tall at 1440px = ~4.0 full viewport scrolls**

- **[Critical]** KPI numbers buried at item 8 — ~900px from top. Users must scroll past Division Health + Recent Audits before seeing the headline conformance score.
- **[Critical]** Tab strip at ~2,800px from top. Tabs serve no scroll-reduction purpose at that depth.
- **[High]** 6 header action buttons — cluttered on desktop, 4 are hidden on mobile creating inconsistency.
- **[High]** Section NC Breakdown cards redundant with Analysis tab charts.
- **[High]** No localStorage persistence for hidden section preferences — sections re-expand on every visit.
- **[Medium]** Division Health card grid at 375px = ~900px for 9 divisions (entire first screen, nothing else visible).
- **[Medium]** "Conformance by Division" chart and "Division Health" cards are the same data shown twice, 600px apart.
- **[Medium]** Filter bar "Compare vs." control is advanced/rarely used but always visible.
- **[Low]** Active filter chip bar adds a third thin visual row (header → filter bar → chip bar) before any data — 3 stacked chrome rows before content.

### Audits List (`/audit-management/audits`)
**Measured: 7,428px tall at 1440px = ~8.3 full scrolls**

- **[High]** The table renders ALL audits without virtual scrolling — the full 7,428px height confirms a very large dataset is being rendered at once. This will cause performance degradation as data grows. The table has a paginator configured in ReportsView but the AuditDashboardView table does not appear to use pagination.
- **[Medium]** The "Delete Selected" button only appears when rows are selected, but its conditional rendering causes the header to reflow when selections change.
- **[Medium]** Row action buttons (Edit / View) are opacity:0 until hover — invisible to users who aren't aware they exist. Mobile users can never discover them.
- **[Low]** Five filter controls (Division, Status, Auditor, From, To) plus 2 buttons — reasonable density on desktop but wraps to 3 rows on 768px.

### New Audit (`/audit-management/audits/new`)
**Measured: 900px = single viewport — no scroll**

- **[Low]** Form has only 1 field (Division dropdown) + 2 buttons. Consider combining this into a modal or slide-over from the Audits List rather than a full route change to improve workflow speed.
- **[Low]** Large empty white space below the 2 buttons. No illustrations, hints, or next-step guidance for new users.

### Corrective Actions (`/audit-management/corrective-actions`)
**Measured: 900px = single viewport (empty state)**

- **[Critical — Data Bug]** All 5 stat counters (Total, Open, In Progress, Overdue, Closed) display "NaN" in both 1440px and 768px views. This is a calculation error — likely the count values are not initialized before the data finishes loading. This is a broken state visible to all users on this page.
- **[Medium]** The "Source" filter dropdown is truncated ("All Sour...") even at 1440px — the column is too narrow. Same for "Status" ("All Stat...").
- **[Low]** Empty state message ("No corrective actions found") appears but the 5 NaN stat cards above it imply data failure, not an empty dataset — confusing messaging.

### Admin Templates (`/audit-management/admin/templates`)
**Measured: 900px = single viewport**

- **[Medium]** Page shows only a "Select division" dropdown with 80% blank space below. No inline hint about what happens after selecting a division, no skeleton/placeholder for the template list. Users may think the page is broken.
- **[Low]** No "Create Template" CTA visible at the top level — it appears only after a division is selected. Consider adding the action button in the header immediately.

### Admin Settings (`/audit-management/admin/settings`)
**Measured: 900px = single viewport**

- **[Low]** Tab strip (Email Routing / Score Targets / User Roles) is clear and well-structured.
- **[Low]** "Deactivate All" button uses orange/warning color which could be confused with a destructive action — consider secondary gray styling.
- **[Low]** The email record shows "—" in the Name column, indicating the name field is empty for the seed user. Should show a fallback like the email address or "Unknown".

### Admin Users (`/audit-management/admin/users`)
**Measured: 900px = single viewport**

- **[Low]** Works cleanly. Stat cards (Total, Active, Disabled) are well-proportioned.
- **[Low]** The "+ Add" role chip in the Roles column is small (estimated ~28px tap target) — below the 44px mobile minimum. Role assignment on mobile would be difficult.
