# EF Core Migration Skill

Adds a new Entity Framework Core migration for a schema change.

## Usage

```
/migrate <MigrationName>
```

Example: `/migrate add_division_audit_frequency`

## What This Skill Does

1. Validates the migration name is provided and follows snake_case convention
2. Runs `dotnet build` on the Data project to catch model errors before generating the migration
3. Generates the migration file using `dotnet ef migrations add`
4. Opens the generated migration for review
5. Reminds you to audit `Up()` and `Down()` before committing

## Migration Naming Convention

- Use snake_case: `add_column_name`, `add_table_name`, `alter_column_name`, `drop_table_name`
- Be specific about what changed: `add_division_audit_frequency_days` not `update_division`
- Prefix with verb: `add_`, `alter_`, `drop_`, `create_`, `remove_`

## Steps Claude Will Execute

```bash
# 1. Restore local tools (ensures correct dotnet-ef version)
dotnet tool restore

# 2. Build the Data project to catch model errors
cd Data && dotnet build --no-incremental

# 3. Add the migration (run from solution root, target Data project)
dotnet ef migrations add <MigrationName> \
  --project Data \
  --startup-project Api \
  --context AppDbContext

# 4. Show the generated migration files
```

## After Migration Is Generated

Claude will:
- Read and display the generated `Up()` and `Down()` methods
- Flag any destructive operations (DropTable, DropColumn, AlterColumn reducing size)
- Verify `Down()` is properly implemented (not just `throw new NotImplementedException()`)
- Check that every FK column has a corresponding `HasIndex` call
- Check that every status column has a `HasCheckConstraint`
- Remind you to run `dotnet build` on the full solution to confirm no compile errors

## Deployment Note

Migrations are **never** auto-applied in Development or Production environments.  
The Azure DevOps pipeline applies migrations using the idempotent SQL script generated by:

```bash
dotnet ef migrations script --idempotent --output migrations.sql \
  --project Data \
  --startup-project Api \
  --context AppDbContext
```

This script is generated in the CI build and applied to the target database before the app is deployed.
